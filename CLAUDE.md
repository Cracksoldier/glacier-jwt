# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

GLACIER — a fully static, client-side JWT/JWE analyzer (decode, WebCrypto signature verification, JWE decryption). Plain HTML/CSS/JS, no build step, no dependencies, no backend, zero runtime network requests.

## Hard constraints (do not break)

- **Must work from `file://` AND GitHub Pages.** Therefore: classic `<script>` tags only — **no ES modules** in `js/` (they fail on `file://` due to CORS). Scripts load in dependency order listed at the bottom of `index.html`.
- **Fully offline:** Font Awesome and all fonts are vendored in `vendor/` — never add CDN links or runtime fetches.
- Only Font Awesome **Free** icon classes work (e.g. `fa-shield-slash` is Pro-only and renders blank — grep `vendor/fontawesome/css/all.min.css` before using a new icon).
- Token/key content and error messages go into `innerHTML` everywhere — always pass dynamic text through `JWTUtils.escapeHtml` (or `highlightJSON`, which escapes internally).

## Commands

```sh
node tools/gen-samples.mjs        # run the whole test suite (~28 checks) AND regenerate js/samples.js
python -m http.server 8741        # serve for browser testing (simulates GitHub Pages)

# headless UI test (27 checks; needs the server above):
"C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --headless=new --disable-gpu \
  --virtual-time-budget=15000 --dump-dom "http://localhost:8741/tools/ui-test.html" \
  | grep -o -E '(PASS|FAIL)  [^<]*'
```

There is no lint/build. Tests live in two places: crypto/logic regressions in `tools/gen-samples.mjs` (Node), DOM/flow checks in `tools/ui-test.html` (headless browser, drives the real page in an iframe). Add regressions for bug fixes to whichever layer can express them. `tools/ui-shot.html?sample=N&act=verify|decrypt&y=PX` is a screenshot harness for visual checks.

## Architecture

Each file in `js/` is an IIFE attaching one namespace to `window` — **or `globalThis` when `window` is absent, which is what lets Node import them for testing** (Node ≥18 has WebCrypto/atob built in). Dependency order:

```
utils.js   → JWTUtils    base64url/bytes, escapeHtml, highlightJSON
parser.js  → JWTParser   cleanToken ("Bearer " stripping), JWS(3-seg)/JWE(5-seg) detection → parse-result object
claims.js  → JWTClaims   claim knowledge base, NumericDate handling, timeStatus (expired/active/...)
keys.js    → JWTKeys     user key material (secret/PEM/JWK) → CryptoKey; KEY_MGMT registry
verify.js  → JWTVerify   JWS verify (HS/RS/PS/ES × 256/384/512)
decrypt.js → JWTDecrypt  JWE decrypt; CBC-HS composed manually (WebCrypto has no native JWE)
analysis.js→ JWTAnalysis security findings derived from a parse result
samples.js → JWTSamples  GENERATED by tools/gen-samples.mjs — never edit by hand
app.js     →             all DOM wiring; everything renders from the parse-result shape
```

The parse-result object returned by `JWTParser.parse()` is the central contract — `app.js`, `analysis.js`, `verify.js`, and `decrypt.js` all consume its fields (`ok/type/header/payload/payloadRaw/signatureBytes/signingInput/jwe.*/errors/warnings`).

Key design points that are easy to violate:

- **`KEY_MGMT` registry in `keys.js`** is the single source of truth for supported JWE key-management algorithms — it drives the import logic, the "Supported: …" error text, and the UI hint via `keyMgmtKind()`. Add algorithms there, not as scattered string checks.
- **Nested-token detection** (`decrypt.js`) delegates to `JWTParser.parse(plaintext).ok` — don't reintroduce a parallel regex; the `cty` header lies in the wild.
- **Async race guard in `app.js`:** `renderSeq` is bumped on every render; `doVerify`/`doDecrypt` capture it before awaiting and drop stale results. Any new async UI action must follow this pattern.
- **JWE AAD** is the ASCII bytes of the *original* base64url protected-header segment (`parsed.jwe.headerB64`), never a re-serialization.
- Timestamps: any finite number is a valid RFC 7519 NumericDate — no sanity windows (an exp of 1973 must report "Expired"). Values > 1e12 get a "looks like milliseconds" finding instead of being ignored.
- Nested-JWE navigation is a stack (`envelopes` in `app.js`), supporting multi-level descent.

## Known platform quirks

- **Chromium rejects 192-bit AES keys** (`A192KW`/`A192GCM`/`A192CBC-HS384` work only in Firefox). `keys.js#aesImportError` converts the opaque `OperationError` into a clear message — keep that path intact. Node *does* support AES-192, so the Node tests cannot catch Chromium-only regressions here.
- Headless Edge on this machine produces corrupted screenshots below ~500px window width; render small assets at ≥512px (this is why `apple-touch-icon.png` is 512×512, not 180).
- SVG gradients need `gradientUnits="userSpaceOnUse"` to paint zero-width geometry (vertical/horizontal lines) — relevant to `favicon.svg`.
